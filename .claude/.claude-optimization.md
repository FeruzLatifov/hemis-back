# Claude AI Optimization Strategy

## Problem
- 5,341 lines of documentation
- 176 KB total size
- Risk: Claude context window overflow
- Solution: Hierarchical structure

## New Structure

### Layer 1: Quick Start (THIS FILE)
**README.md** - 100 lines
- Golden rules
- Daily commands
- Quick checklists
- Links to detailed docs

### Layer 2: Core Rules (ALWAYS READ)
**rules.md** - Keep concise, focus on:
- Golden rules
- Module-specific rules
- Quick reference tables
- Code examples (minimal)

### Layer 3: Detailed Guides (READ WHEN NEEDED)
**Separate files:**
- SWAGGER_GUIDE.md (extract from MANDATORY_REQUIREMENTS.md)
- TESTING_GUIDE.md (extract from MANDATORY_REQUIREMENTS.md)
- LIQUIBASE_GUIDE.md (already separate ✓)

### Layer 4: Reference (RARELY READ)
- architecture.md (system design)
- context.md (project context)

## Benefits
1. Claude reads less initially
2. Faster response time
3. Can request specific guide when needed
4. Easier to maintain
5. Better organization

## Recommendation
Keep total "always read" docs under 2000 lines:
- README.md: ~100 lines
- rules.md: ~800 lines (streamlined)
- context.md: ~500 lines (essential only)
Total: ~1,400 lines (✓ under 2000)

Detailed guides: On-demand only
